# Better Colony Automation (BCA) - 殖民地自动化改良 (4.3)

[4.2旧版本](https://steamcommunity.com/sharedfiles/filedetails/?id=3601183337)

由于罪恶的P社在4.0更新后一直没修复自动托管的bug，导致每到中后期的游戏体验极差，最后还得是自己来解决P社的bug

反馈与交流：QQ 153870310

快速链接 [Stellar AI 兼容性模组](https://steamcommunity.com/sharedfiles/filedetails/?id=3669715825)

## 快速开始

1. **全局设置**：在【政策】中设置**自动建造矿物预留**。
2. **开启自动化**：在行星界面开启【行星自动化】以及对相关自动化选项进行配置。
3. **混合规划与区划建造设置** 现在已提供行星 UI 界面来设置混合规划和区划建造。
4. **杂项设置** 目前仍有部分设置需要通过行星决策来调整，例如焕活的建设限制。

## 总览(v1.1 已发布!)

### 建造功能

#### 自动建造逻辑
- **按需建造**：仅在岗位不足、且星球不存在建设中建筑的情况下才进行建造。可以通过设置进行无视岗位需求的提前建造
- **特殊建筑**：人口增长相关的建筑（诊所、克隆舱、焕活所、基因研究所），以及增益型建筑（如生产部等）在有岗位空余时也会建造
- **矿物预留**：增加了国家级的**矿物预留政策**。你可以设置矿物阈值，若国家矿物低于该值，自动化将暂停建设，避免造成经济崩溃。

#### 增加自动化选项
- **医疗**：支持基础医疗建筑、基因组研究所及分子焕活所的建设。
- **文化丰碑**：自动建造凝聚力产出建筑。
- **资源仓储**：自动建造仓库。
- **人口管理细化**：（非机械帝国）可独立管理机器人组装厂的建设。
- 移除了避免财政赤字的选项，若不手动关闭会经常卡住建筑建设。

#### 混合规划管理

之前的混合规划星球类型被移除，现在采用GUI界面来进行混合规划设置

注意（次要区划混合规划设置），本MOD在存在相同的次要区划（理想城/机械世界/蜂巢星球/环世界）时无法保证实际特化的槽位和设置的槽位对应，需要手动调整设置来使其与实际建设的特化对应。否则会导致区划建设数量和图标不对应。

#### 建筑建设
优先建造辅助性建筑，如诊所、克隆舱、文化丰碑、生产部等、以提升星球整体效能，主产出建筑在辅助建筑建造完成后再进行建设。
本模组对原版的自动建造逻辑进行了大量改良，包括适配各种特殊星球类型，稀有资源建筑的建造逻辑也得到了修复。

- **仓库建设**：默认只建造一个仓库。可以通过GUI设置来建造更多仓库，其还会自动建造一产区块以容纳仓库建筑。
- **分子焕活所**：默认只建造一个焕活所。可以通过行星决策来解除限制。

#### 区划建设

可通过GUI界面设置相关建设规划，模组会根据设置自动进行区划建设以达到规划要求。（暂时没做拆除，因为手动替换的差别不大）

### 拆除功能

#### 建筑拆除
- **职能不符拆除**：当建筑槽位已满且当前建筑不符合星球当前的专精或混合规划时，模组会自动将其移除。例如：在科研球上有工厂且没地方造实验室时，工厂会被拆除。
- **人口组装拆除**：当星球彻底填满（无住房、无岗位、无剩余空位）时，人口组装建筑会被自动拆除，为更高级的产出建筑腾出空间。可以通过政策关闭此功能。

#### 区划拆除

- **一产星球**：在自动区划管理模式下，会自动进行区划拆除与建造以最大化产出。
- **理想城计划**：当理想城候选规划设置开启时，模组会自动拆除一产区划。

#### 区划特化拆除

**不在GUI界面规划内的特化均会被拆除**

### 高级建设计划

目前支持一产星球的高级建设计划，高级建设计划包含两个部分

1. **最优建设**：自动选择在**最短时间内带来最大产出**的建筑/区划/特化进行建设
2. **区划替换**：在区划已经建设满后，若替换区划后能带来产量提升，则会进行区划拆除与建设

高级建设计划的区划管理可通过gui设置关闭

### 其他

- **理想城候选规划设置**：当可以进行理想城建设时可开启此设置，允许提前规划理想城的建设。开启后将自动拆除一产区划，并在满足建设条件时提示玩家是否正式开启项目。
- **去城市化决策**：测试用途。

## 兼容性

- 本模组对的**建筑拆除**功能对原版建筑进行了修改，可能会与修改原版建筑的模组产生冲突。建议在使用本模组时避免加载其他修改原版建筑的模组。如果你能容忍没有建筑拆除功能，可以在加载顺序上将本模组放在修改原版建筑的模组之前。
- 修改ai行为的模组可能会与本模组产生冲突，建议将本模组放在加载顺序的最后。
- 本模组只支持原版建筑和区划的自动托管，暂不支持任何来自其他模组的建筑和区划。
